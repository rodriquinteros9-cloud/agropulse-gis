Implementacion del backend (FastAPI + GEE)3.1 Nuevo endpoint requeridoCrear el siguiente endpoint en el backend FastAPI:POST /api/lote/imagenes-ndviRequest body (JSON):{ "geometry": { ...GeoJSON Polygon... }, "fecha_inicio": "2024-01-01", "fecha_fin": "2024-12-31", "max_nubes": 25, "indice": "NDVI" }Response (JSON):{ "imagenes": [ { "fecha": "2024-10-03", "fecha_iso": "2024-10-03T00:00:00Z", "nubes_pct": 1, "ndvi_promedio": 0.72, "ndvi_min": 0.18, "ndvi_max": 0.91, "tile_url": "https://earthengine.googleapis.com/...", "thumbnail_url": "https://earthengine.googleapis.com/..." }, ... ] }3.2 Logica GEE del endpointUsar la coleccion COPERNICUS/S2_SR_HARMONIZED de Google Earth Engine. La logica del endpoint debe seguir estos pasos en orden:Autenticar GEE con las credenciales del proyecto (ya configurado en el repo via .env / earthengine credentials).Convertir el GeoJSON recibido en una ee.Geometry.Polygon.Filtrar la coleccion: .filterBounds(geometry).filterDate(fecha_inicio, fecha_fin).filter(ee.Filter.lt('CLOUDY_PIXEL_PERCENTAGE', max_nubes)).Para cada imagen de la coleccion, calcular el indice seleccionado. Para NDVI: ndvi = imagen.normalizedDifference(['B8', 'B4']).rename('NDVI'). Para EVI usar la formula estandar con B8, B4, B2. Para NDRE: normalizedDifference(['B8', 'B5']).Recortar la imagen al poligono del lote con .clip(geometry).Calcular estadisticas del lote: reduceRegion con ee.Reducer.mean(), ee.Reducer.min(), ee.Reducer.max() sobre la banda del indice.Generar tile_url usando imagen.getMapId() con la paleta RdYlGn y los min/max del lote (normalizacion intralote para el modo 'contrastado').Generar thumbnail_url usando imagen.getThumbURL() con dimensiones 80x58px y la misma paleta.Ordenar los resultados por fecha descendente y devolver el array.3.3 Paleta de colores NDVIUsar siempre esta paleta RdYlGn de 7 pasos como string para GEE:palette = ['d73027', 'f46d43', 'fdae61', 'ffffbf', 'd9ef8b', 'a6d96a', '1a9850']Para NDVI estandar: min=-0.2, max=0.9. Para NDVI contrastado: usar el ndvi_min y ndvi_max calculados del propio lote (estiramiento de contraste intralote).3.4 Manejo de errores y performanceSi GEE devuelve un error de timeout (tipico en lotes grandes), responder con HTTP 503 y mensaje claro.Implementar el endpoint como async usando asyncio para no bloquear el servidor FastAPI durante las llamadas a GEE.Agregar CORS para el origen del frontend (ya configurado en el repo, verificar que el nuevo router lo hereda).Limitar el response a un maximo de 50 imagenes. Si hay mas, tomar las 50 mas recientes.4. Implementacion del frontend (React + TypeScript + Leaflet)4.1 Nuevo componente principalCrear el componente SatelliteViewer.tsx en la carpeta de componentes existente del repo. Este componente recibe como prop el lote activo (con su geometria GeoJSON y metadata) y renderiza toda la UI del visor.4.2 Estructura de subcomponentesSatelliteViewer.tsx — componente raiz, maneja el estado global de la featureSatelliteTopbar.tsx — topbar con nombre del lote, selector de indice y badge de nubesSatelliteMap.tsx — wrapper del mapa Leaflet con la capa raster NDVI y los overlaysSatelliteFilmstrip.tsx — tira horizontal de miniaturas con scrollFilmstripCard.tsx — tarjeta individual del filmstrip4.3 Estado y flujo de datosEl estado del componente SatelliteViewer debe manejar:imagenes: array de objetos ImagenSatelital (tipado con TypeScript interface)imagenActiva: index o ID de la imagen seleccionada en el filmstripindiceSeleccionado: 'NDVI' | 'NDVI_C' | 'EVI' | 'NDRE'loading: boolean — true mientras se cargan las imagenes del backenderror: string | null — mensaje de error si el fetch fallaAl montar el componente (o al cambiar el lote activo), disparar el fetch al endpoint POST /api/lote/imagenes-ndvi con la geometria del lote y los parametros por defecto. Al cambiar el indice seleccionado, refetch con el nuevo indice.4.4 Capa raster en LeafletPara mostrar el heatmap NDVI sobre el mapa usar L.tileLayer() de Leaflet con el tile_url retornado por GEE. Cada vez que el usuario selecciona una imagen distinta en el filmstrip, remover el TileLayer anterior y agregar el nuevo. El TileLayer de GEE se agrega por encima del basemap (OpenStreetMap o ESRI Satellite) y por debajo del vector del poligono del lote.El contorno del poligono del lote debe mantenerse siempre visible encima del raster, con stroke blanco de 2px y fill transparente.4.5 TypeScript interfaceDefinir la siguiente interface en un archivo de tipos compartido (types/satellite.ts o similar):interface ImagenSatelital { fecha: string; fecha_iso: string; nubes_pct: number; ndvi_promedio: number; ndvi_min: number; ndvi_max: number; tile_url: string; thumbnail_url: string; }4.6 Estilos y diseñoEl componente debe integrarse al sistema de estilos existente en el repo (Tailwind CSS). Restricciones y consideraciones importantesNo instalar librerias nuevas en el frontend sin verificar primero si hay una alternativa ya disponible en el package.json del repo. Leaflet ya esta instalado — usarlo directamente.No cambiar la configuracion de CORS ni de autenticacion GEE existente en el backend. Solo agregar el nuevo router/endpoint.Las credenciales de GEE se montan via volumen Docker desde ~/.config/earthengine — no hardcodear ni modificar esa logica.El tile_url de GEE tiene expiracion de tiempo. No cachear ese URL por mas de 1 hora en el frontend.Los calculos de estadisticas (mean, min, max) de GEE pueden tardar 3-8 segundos para lotes grandes. El endpoint debe tener un timeout configurado de 30 segundos y comunicarlo al frontend.Para el modo NDVI contrastado, el min/max de la paleta debe calcularse por imagen, no por lote completo. Esto significa una llamada a reduceRegion por imagen antes de generar el tile_url.8.
